ONU注册与查看相关命令

查看未注册ONU信息
show pon onu uncfg
// 查看OLT中未注册ONU信息
查看定位LOID在PON网络中的位置信息
show gpon onu by loid XXXXXX
// 通过光猫的LOID号XXXXX，可以查询到其在PON网络中的具体位置信息
查看ONU上线记录及串码SN号
show gpon onu detail-info gpon_onu-1/2/1:1
// 查看ONU近10次的上线记录及串码SN号,故障查修判断时可能用到
查看PON口配置
show running-config-interface gpon_olt-1/3/6
// 查看OLT的1/3/6槽位注册ONU数量
查看ONU注册详情数据
show running-config-interface gpon_onu-1/3/6:2
// 显示PON口下ONU的运行配置详情数据,深入了解ONU的网络配置情况
查看ONU的VLAN配置数据
show running-config-interface vport-1/3/6.2:1
// 查看ONU业务流配置以及组播接收端口配置
同OLT内，PON之间的业务割接场景很常见，涉及pon-copy命令
1.设置源PON口模式为mode1
interface gpon_olt-1/2/1
resource-id-assign-mode mode1
2.退出config模式然后write保存下
3.执行pon-copy命令,在config模式下
pon-copy gpon olt interface gpon_olt-1/2/1 gpon_olt-1/2/8
4.设置目的PON口模式为mode2
interface gpon_olt-1/2/8
resource-id-assign-mode mode2
5.退出config模式然后write保存下
// 表示将1/2/1PON口下的所有业务数据copy到1/2/8PON口下面了
查看PON口下onu详细运行状态
show gpon onu state gpon_olt-1/2/1
// 查看PON口下onu详细运行状态
💡 提示： 注：参数“Phase State”可能有如下几种取值： （1） offline，表示该 ONU 属于离线配置，OLT 没有搜索到这个 ONU； （2） logging，表示该 ONU 已经搜索到，正在测距； （3） syncMib，表示该 ONU 已经测距完毕，正在同步数据； （4） working，表示 ONU 同步结束，可以进行其他配置； （5） LOS，表示 ONU 和 OLT 之间的光纤链路有故障，导致 ONU 掉线； （6） DyingGasp，表示 ONU 断电了，可能是 ONU 正常被关电，也可能是 ONU 所 在位置市电断了；
